# 视觉感知方案选型

## 项目目标

- ###### 在确保接触力恒定且绝对不损伤模型（未来是活体羊）的前提下，让机械臂沿羊背部曲面完成高覆盖率的剃毛作业；系统需具备对表面几何的实时感知能力，以应对目标轻微运动和环境干扰，并为未来部署到动态、非结构化的真实场景奠定鲁棒性基础。

- 当前研究阶段目标:

  ```
  - 只剃背部，允许少量重复/遗漏；
  - 羊模型是刚性的；
  - 推子需紧贴并施加压力；
  - 接受角度偏差，但接触力必须恒定，不能划伤模型；
  - 当前用于研究，最终要部署到真实活羊场景；
  - 失败包括：划伤、超时、大块区域漏剃。
  ```

## 关键约束和假设

- 覆盖路径生成:

  ```
  计划先做离线方案：先配准完整背部点云 → 规划路径 → 执行时仅依赖预规划轨迹 + 力控。
  ```

- 建模阶段:

  ```
  - 羊不动，但不能手动转羊或相机（要自动化）；
  - 执行阶段暂不使用视觉，只靠力控跟踪预规划轨迹；
  - 希望几秒内从采集到执行，为实时方案打基础。
  ```



## 核心挑战识别

- 深度相机看到的是“毛表面”，但操作面是“近皮肤层”

  ```
  点云 ≠ 皮肤，但可通过安全余量 + 力控补偿，不必毫米级精确
  ```

-  配准依赖多视角重叠，但机械臂可能遮挡固定相机

  ```
  侧前方固定安装易被 RM65-B 本体遮挡 → 重叠率低
  ```

- 需平衡：成本 vs 速度 vs 可扩展性

  ```
  眼在手外（多相机）快，但成本高；眼在手上成本低，但需机械臂移动，采集慢。
  ```

  

## 视觉方案选型分析

### 眼在手上

#### 优点:

```
- 自包含，无需改造环境；
- 同一套硬件可平滑过渡到在线系统；
- 支持主动感知与数据记录；
- 成本最低，失败代价小；
- 采集时间可通过优化控制在 5 秒内，满足研究需求。
```

#### 缺点:

```
- 采集较慢（需机械臂移动）。
```

#### 适用性:

```
实验室快速迭代、探索性研究。
```



### 眼在手外(相机阵列)

#### 优点:

```
- 采集快、点云稳定。
```

#### 缺点:

```
- 依赖工场基建、易被机械臂遮挡（侧装）、成本高。
```

#### 适用性:

```
标准化产线部署阶段
```



### 混合方案

```
标定复杂度高，当前阶段不推荐。
```



## 决策:眼在手上方案

#### 理由:

```
符合实验室“低成本、快迭代”需求；系统自包含，无需改造环境；
支持从离线 → 在线平滑演进；
可主动控制视角，确保配准成功率；
采集时间可通过优化控制在 5 秒内，满足研究需求。
```





## 修改:眼在手外+眼在手内

>眼在手外负责粗略分区,眼在手内负责局部精细的点云处理,
>
>这样就不需要配准,而是直接利用眼在手外的分区进行处理,
>
>且眼在手外的工作使他对精度的要求不高

>前期可先暂时用眼在手上,效果不够再使用这个方案,比较循序渐进
